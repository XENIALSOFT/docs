# X-Accel

## 핵심 개념

사용자의 인증이나 권한 검사 등의 비즈니스 로직은 백엔드 애플리케이션(Spring Boot, Nuxt BFF 등)이 처리하고, 검증이 완료된 대용량 파일(이미지, PDF, 영상 등)의 실제 전송은 정적 파일 처리에 압도적으로 뛰어난 `Nginx`에게 위임(Internal Redirect)하는 기능입니다.

백엔드 서버가 대용량 파일 스트리밍까지 도맡아 처리하면 서버 메모리와 CPU가 고갈되어 다른 API 요청을 처리하지 못하는 병목 현상이 발생합니다. `X-Accel`은 이 무거운 짐을 `Nginx`에게 넘겨 백엔드의 부담을 최소화합니다.

## 동작 원리

1. 클라이언트 요청: 사용자가 권한이 필요한 파일을 요청합니다.
2. 백엔드 검증: 요청이 `Nginx`를 거쳐 백엔드로 들어갑니다. 백엔드는 세션이나 JWT를 확인하여 이 사용자가 파일을 다운로드할 권한이 있는지 체크합니다.
3. 특수 헤더 반환: 권한이 확인되면 백엔드는 파일 데이터를 직접 읽어서 응답하지 않고, `Nginx`만 알아볼 수 있는 특수 헤더인 `X-Accel-Redirect: /protected/files/document.pdf`를 붙여 **0바이트짜리 빈 응답**을  `Nginx`에 보냅니다.
4. 내부 리다이렉트 및 전송: 헤더를 가로챈 `Nginx`는 클라이언트 몰래 내부적으로 지정된 경로의 실제 파일을 디스크에서 읽어 사용자에게 초고속으로 전송합니다.(클라이언트는 내부에서 이런 일이 일어났는지 알 수 없습니다.)

### nginx 설정

`Nginx` 설정에서 가장 중요한 포인트는 외부 사용자가 접근할 수 없는 보안 구역(`internal`)을 만드는 것입니다.

```Nginx
# nginx.conf 또는 사이트 설정 파일

server {

    ...nginx 설정

    # X-Accel-Redirect를 통해 내부적으로만 접근 가능한 가상 경로
    location /protected/ {
        internal; # 외부에서 직접 /protected/로 접속하면 404를 반환함!
        
        # 실제 파일이 물리적으로 저장되어 있는 서버의 디렉토리 경로
        alias /var/www/data/storage/; 
    }
}
```

### 마무리

`X-Accel`은 "보안(애플리케이션 영역)"과 "성능(Nginx 고성능 정적 파일 처리)"이라는 두 마리 토끼를 모두 잡을 수 있는 영리한 아키택처 패턴입니다.
